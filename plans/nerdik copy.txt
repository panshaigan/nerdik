Privacy/data: [FUTURE]
	- Requirement: Implement account deletion that either deletes or anonymizes personal data, while preserving system integrity (past activities/events).
	- Requirement: Provide a machine‑readable export (JSON/ZIP) of user‑related data on demand.
	- Request logs with minimal PII (IP, user_id if logged in, route).
	- DB audit logs on critical entities: event, organization, activity, activity_proposal, user profile changes, tags.
	- Only a small set of admin users can access detailed logs and user data; every admin action on user data (viewing profile, changing status, banning, editing content) should be recorded with timestamp and admin id.
	- Log retention is limited (e.g. 12 months) unless needed for legal reasons.

Hosting notes
	- For dev:
		- Local Docker / Laravel Sail.
	- For first public deploy:
		- One small VPS (Hetzner/Contabo/etc):
		- Nginx + PHP‑FPM + MariaDB (same VM) or docker
		- Supervisor for queue workers / schedulers.
		- Let’s Encrypt for HTTPS.
		- CI/CD via GitHub Actions & SSH.
	- Once the app grows, you can:
		- Split DB to a managed MariaDB.
		- Add another VPS for queues or for future mobile/API scaling.
